How a Computer Works

Learning Hardware Systems Architecture Through the Nintendo Game Boy

Why Study the Game Boy?

Modern computers run billions of operations per second across layers of abstract operating systems, making it difficult for students to visualize real hardware behavior. The original 1989 Game Boy (DMG-01) operates exactly like a modern machine but at an accessible, human scale. It features a CPU, explicit memory domains, standard buses, and registers that can be fully map-analyzed in a single classroom lab loop.

1 The Brain: The CPU Sharp LR35902

The Central Processing Unit (CPU) is the logical engine driving the computer. Running at 4.19 MHz, its internal quartz crystal oscillates 4.19 million times per second. It fetches low-level assembly instruction symbols, decodes them, and coordinates physical operations across registers.

Registers

Eight 8-bit registers can be paired into 16-bit configurations for specific address operations:

  • Accumulator (A): Holds results for arithmetic and logical operations.
  • Flags (F): Tracks CPU state flags.
  • BC, DE, HL: General purpose pairs. HL points directly to memory addresses.
  • SP & PC: 16-bit Stack Pointer and Program Counter.

Flags Register (F)

The lower 4 bits stay at zero. The top 4 bits signal execution status:

  • Bit 7 - Zero (Z): Set if an operation result equals 0.
  • Bit 6 - Subtraction (N): Set if the previous operation was a subtraction.
  • Bit 5 - Half Carry (H): Set if a lower nibble arithmetic overflow occurs.
  • Bit 4 - Carry (C): Set if an operation overflows bit 7 or borrows past 0.

Instruction Set Groups

  • Loads: Move data between registers or memory (LD A, B, LD ($C000), A).
  • Arithmetic/Logic: Compute values inside the ALU (ADD A, B, XOR A).
  • Bit Ops: Test, set, or clear specific register bits (BIT 7, H, SET 3, L).
  • Rotates/Shifts: Shift bits left or right for fast binary multiplication (RLA).
  • Control Flow: Jump or call subroutines conditionally (JP NZ, $0150, CALL $2000).

2 The Router: The MMU & Memory Map 16-bit Address Bus

The system features a 16-bit wide address bus line, allowing the CPU to point directly to exactly 65,536 unique storage locations ($2^{16} = \text{64 KB}$). To manage everything with limited wiring, the Game Boy uses Memory-Mapped I/O: everything from game cartridges to controller button matrices shares this single linear map zone.

Address Range Hardware Allocation Component Functional Mechanics
0x0000 - 0x7FFF Cartridge ROM Static game storage code. Fixed area + bank switching window.
0x8000 - 0x9FFF Video RAM (VRAM) Dedicated block for graphical background tile arrays.
0xA000 - 0xBFFF External Cartridge RAM Optional volatile workspace, often battery-backed for saves.
0xC000 - 0xDFFF Work RAM (WRAM) The main volatile scratchpad memory for game state arithmetic variables.
0xFE00 - 0xFE9F Object Attribute Memory (OAM) Fast access tracking table storing coordinates for up to 40 sprites.
0xFF00 - 0xFF7F I/O Hardware Registers Direct logic ports linking memory access to peripheral chips.

3 Bank Switching: Memory Bank Controllers MBC Chipsets

Since many games are larger than 32 KB, cartridges contain an integrated chip called an MBC (Memory Bank Controller). When the CPU needs data from a distant part of the cartridge, it writes a bank index number to an external register, switching the layout visibility instantly.

ROM ONLY

Up to 32 KB ROM capacity. Maps directly into the system bus layout. Does not support bank switching registers or external RAM chips.

MBC1

Up to 2 MB ROM and 32 KB RAM. Splitting fixed bank 0 and a switchable bank window. Writing to 0x2000-0x3FFF toggles the active ROM bank index.

MBC2

Up to 256 KB ROM. Contains a built-in internal RAM workspace consisting of exactly 512 half-bytes (4-bit nibbles). No external RAM chip support.

MBC3

Up to 2 MB ROM and 64 KB RAM. Adds hardware support for an integrated Real-Time Clock (RTC) chip to latch tracking times directly via the memory bus layout.

MBC5

Up to 8 MB ROM and 128 KB RAM. Resolves mapping design quirks found in MBC1 by allowing bank 0 to map cleanly into the switchable slot. Uses two split configuration registers to achieve full 9-bit bank selection addresses.

4 The Painter: Graphics VRAM & PPU Picture Processing Unit

The Game Boy does not use a modern layout frame-buffer where pixels are written to a grid matching the screen resolution. Instead, the PPU parses structural tile matrices located inside the VRAM range (0x8000 - 0x9FFF) and maps them dynamically onto the LCD panel synchronized with physical horizontal and vertical scanline loops.

VRAM Data Layout

  • Tile Data (0x8000 - 0x97FF): Stores the raw 8x8 pixel tile bitmaps. Every pixel uses 2 bits to achieve a 4-shade grayscale palette. Each tile line consumes exactly 2 bytes (interleaved "2bpp" format).
  • Background Maps (0x9800 - 0x9FFF): Contains two 32x32 index arrays. These arrays reference the tile blocks above to assemble a massive 256x256 pixel virtual background plane.

PPU Hardware Modes

As the PPU draws the active screen resolution (160x144 pixels), it steps through four sequential micro-operation loops tracked inside register 0xFF41 (STAT):

  • Mode 2 (OAM Search): Cycles 0-80. The PPU scans the Object Attribute Memory to find up to 10 sprites matching the current horizontal scanline.
  • Mode 3 (Pixel Transfer): Cycles 81-252+. The PPU pulls layout bitmap details from VRAM and streams pixels directly to the display logic.
  • Mode 0 (H-Blank): Remaining cycles of the line. Temporary idle period after drawing a row. The CPU regains full read/write permissions to VRAM/OAM.
  • Mode 1 (V-Blank): Triggered once all 144 rows are drawn. Lasts for 10 structural scanlines. Fires a high-priority hardware interrupt for frame logic resets.

Classroom Connection: This tile-based rendering design maximizes graphics processing output with minimal physical system memory footprint. By modifying metadata attributes inside the fast-access OAM table (0xFE00 - 0xFE9F)—such as X/Y positions, tile indexes, layering priority, or flip flags—programmers can move complex hardware sprites across coordinates instantly without losing cycles redrawing or erasing individual background pixel fields.

5 The Synthesizer: Sound APU Audio Processing Unit

The Game Boy does not play high-definition audio files. It is a true hardware audio synthesizer. It reads register inputs from the 0xFF10 memory block to oscillate electrical signals directly into 4 distinct physical audio channels:

Pulse 1

Square wave generator featuring sweep hardware. Used for dynamic movement sound effects.

Pulse 2

Standard square wave channel. Used for primary melodic composition sequences.

Wave Ch.

Reads a tiny 32-sample custom buffer array. Used to produce synthesized basslines.

Noise Ch.

Generates pseudo-random binary white noise sequences. Handles explosions and percussion beats.

6 The Clockwork: Hardware Timer & Interrupts TIMA Register

Computer loops require stable timing mechanisms to ensure identical performance across variable frame conditions. The Game Boy features a hardware divider network counting down to zero. By configuring memory register 0xFF05 (TIMA), systems run background processing independently from display operations.

Classroom Connection: When the timer counter rolls over from its maximum capacity value (255 to 0), it triggers a Hardware Interrupt Request (IRQ). The CPU immediately halts its current function thread, stores its register states, executes the timing module, and resumes operations without drop frames.

7 The Gateway: Joypad Matrix Input Register 0xFF00

To reduce trace lines and save physical pins on the main circuit board, the 8 button paths are mapped into a single $2 \times 4$ logic matrix array managed at address 0xFF00.

[CPU Line Selection Out] / \ Bit 4 Low: Directions Bit 5 Low: Action Keys | | ➔ Bit 3: Down ➔ Bit 3: Start ➔ Bit 2: Up ➔ Bit 2: Select ➔ Bit 1: Left ➔ Bit 1: Button B ➔ Bit 0: Right ➔ Bit 0: Button A

Classroom Connection: To read user input, the game loop switches bit states inside register 0xFF00 to select either the direction or action row, then reads the lower 4 data bits back to compute movement matrices via standard bitwise AND masks.
